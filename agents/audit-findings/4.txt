4.1.4. Double Credit Diﬀerent Targets
Unit Name: DynamicCurrencyDelta
Location: contracts/evm/src/libraries/DynamicCurrencyDelta.sol in syncBalanceAsCredit function,
applicable when called with diﬀerent target addresses but the same owner
Description: The syncBalanceAsCredit function reads the owner 's ERC20 balance via
currency.balanceOf(owner) and credits it to the target 's delta without consuming, transferring,
or locking the underlying tokens. Because the owner 's balance remains unchanged after
each call, calling syncBalanceAsCredit with the same owner but diﬀerent target addresses
credits the same token balance to each target independently:
1. owner holds 1000 tokens, target1 delta = 0
2. syncBalanceAsCredit(currency, owner, target1) → target1 delta becomes +1000
3. owner balance is still 1000 (no transfer occurred)
4. syncBalanceAsCredit(currency, owner, target2) → target2 delta becomes +1000
5. Total credits: 2000, actual backing: 1000
The sync and syncPair external functions on VTSCurrencyDelta accept arbitrary owner and
target parameters (protected only by _assertBoundFactoryCaller ). Any protocol-bound caller (a
bounds-approved address) could invoke sync with the same balance-holding owner but
diﬀerent target addresses, inflating total credits beyond the actual token custody. This
creates phantom credits that, when taken via subsequent take calls, would drain actual
tokens from the system.
The confirmed PoC at confirmed_pocs/DynamicCurrencyDelta_double_credit_same_balance.t.sol
demonstrates this exact scenario: the same 1000-token balance is credited to 10 diﬀerent
targets, creating 10,000 in total credits from only 1,000 in actual tokens.
User Verdict: True Positive
