# Autocover Tests

This directory contains automated unit tests generated by **Olympix dev tooling** to deliver comprehensive last-mile test coverage of the Fiet protocol.

The test files in `autocover` omit core unit tests produced by the Usher Labs team. Autocover tests leverage the `tools/OlympixUnitTest` base contract utility to indicate intent to generate via Olympix.

## Purpose

These tests are automatically generated to ensure that critical code paths, edge cases, and error conditions across the protocol are thoroughly tested. They complement the manually written integration and end-to-end tests by providing focused unit-level coverage.

## Test Base Contract

All autocover tests inherit from `OlympixUnitTest`, which provides a standardised testing interface for the Olympix tooling infrastructure.

## Running Tests

Run all autocover tests:

```bash
forge test --match-path "test/autocover/**/*.t.sol" -vvv
```

Run a specific test file:

```bash
forge test --match-path test/autocover/modules/PositionManagerImpl.t.sol
```

## Maintenance

These tests are generated and maintained by Olympix dev tooling. When making changes to the protocol:

1. **Do not manually edit** these test files unless fixing a bug in skeletons or with intent to regenerate via the `olympix` CLI.
2. Regenerate tests using the Olympix tooling after significant protocol changes. [Learn more](https://olympix.github.io/CLI/Unit%20Testing/)
3. Report any issues with test generation or coverage gaps to the development team

## Coverage Goals

The autocover tests aim to achieve:

- High code coverage across all protocol contracts
- Edge case validation for critical functions
- Error condition testing for all revert paths
- Boundary condition testing for numeric operations
